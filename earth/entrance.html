🤓 Pour les curieux : les détails techniques

Architecture du Protocole N² :

🔄 Synchronisation automatique : Votre station Astroport interroge régulièrement (toutes les heures) les relais Nostr pour récupérer les nouveaux événements de votre constellation N². Les données sont stockées localement et redistribuées via des tunnels IPFS P2P sécurisés.

Types d'événements synchronisés (NIPs) :

  • Kind 0 (NIP-01) : Profils utilisateurs (métadonnées, avatars, bio)
  • Kind 1 (NIP-01) : Notes et posts de microblogging
  • Kind 3 (NIP-02) : Listes de contacts (follow graph)
  • Kind 7 (NIP-25) : Réactions (likes, emojis) - monétisés en Ẑen
  • Kind 21/22 (NIP-71) : Événements vidéo
  • Kind 1111 (NIP-22) : Commentaires (avec threading)
  • Kind 30023 (NIP-23) : Articles de blog format long
  • Kind 30024 : Événements calendrier

Stack technologique :

📡 Protocole Nostr
Messagerie décentralisée résistante à la censure (NIP-01 à NIP-71)
🔒 Tunnels IPFS P2P
Connexions chiffrées peer-to-peer entre nœuds Astroport
⏰ CRON Jobs
Synchronisation horaire automatique (backfill_constellation.sh)
🔍 Découverte N²
Expansion naturelle du réseau (amis d'amis) via Kind 3

🛡️ Résilience : Pas de point de défaillance unique. Si une station tombe, votre réseau continue de fonctionner. Les données sont redondées sur plusieurs nœuds de la constellation via IPFS swarm privé. Chaque station peut servir de relais de secours.
