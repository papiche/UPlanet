Entrez une adresse IPFS pour explorer son contenu
Chaque ressource IPFS est identifiée par l'empreinte cryptographique de ses données — son CID. Ce navigateur lit la structure de répertoire directement depuis les blocs du réseau, sans serveur intermédiaire, via ?format=dag-json.
🔬 Nœud DAG-PB brut

Réponse exacte du gateway avec Accept: application/vnd.ipld.dag-json — le répertoire UnixFS exposé comme nœud Merkle DAG : 


    

      Chaque Links[].Hash est le CID exact de l'enfant.
      Modifier un seul octet dans n'importe quel fichier change le CID de tout le chemin parent —
      c'est la garantie d'intégrité du Merkle DAG.
    

  


  
  

    
⚙️ Comment ça marche

    

      Un répertoire IPFS est un nœud DAG-PB (Directed Acyclic Graph — Protocol Buffers / UnixFS).
      Il contient une liste de Links vers ses enfants, chacun identifié par son propre CID.


      Ce navigateur envoie GET /ipfs/<cid>/<chemin>?format=dag-json
      au gateway local, reçoit le JSON du nœud, extrait le champ Links
      et affiche les entrées — sans bibliothèque externe.


      Pour les uDRIVE UPlanet, un manifest.json enrichit la vue
      avec miniatures, descriptions IA et métadonnées. L'identité NOSTR signe les publications via IPNS.
